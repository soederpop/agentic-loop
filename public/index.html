System Overview

You define a vision. You establish goals. You record ideas. You work with an AI assistant to align your ideas to your goals and gain clarity. Then, your ideas are turned into projects that can be built and deployed into the world — automatically, through an agentic loop that runs in the background.

Built on the Luca framework (Lightweight Universal Conversational Architecture), powered by Bun runtime, with a Contentbase ORM for organized structured markdown collections.

Authority Process
luca main — port 4410

The Central Hub

Long-running singleton process that orchestrates all subsystems. Uses a lock to ensure only one instance runs system-wide.

Core Services
  • WebSocket server for client connections
  • Git status tracking & broadcasting
  • Log streaming (200-entry circular buffer)
  • Remote console REPL access
Managed Subsystems
  • task-scheduler background task loop
  • project-builder plan executor
  • voice-service voice pipeline
  • presenter native window manager
Operating Modes
  • Authority — First instance: runs the WebSocket server and all subsystems
  • Client — Subsequent instances: connect and show a live TUI dashboard
  • Console — --console flag: REPL connection for debugging
Controls
  • luca main --pause pause the agentic loop
  • luca main --resume resume operations
  • luca main --stop graceful shutdown of subsystems
  • luca main --kill full process shutdown
Document Model
Contentbase ORM — docs/

Structured markdown with YAML frontmatter. Validated by cnotes validate. Each type lives in its own folder with enforced naming and heading patterns.

Goal goals/
High-level objectives. Horizons: short, medium, long. Tracks success criteria and motivation.
Idea ideas/
Initial concepts. Status: spark → exploring → ready → promoted. Links to a Goal.
Project projects/
Full specifications with overview, execution order, and linked Plans.
Plan plans/
Detailed work items for Claude Code sessions. Tracks cost, turns, tool calls.
Play plays/
Repeatable scheduled prompts. Schedule: daily, hourly, 4pm, every-half-hour.
Task tasks/
One-off automation tasks. Executed once, marked completed with timestamp.
Prompt prompts/
Reusable templates with dynamic input variables. Can be repeatable.
Report reports/
Long-form writeups linked to Goals with tags for organization.
Memory memories/
Assistant context: SELF.md, USER.md, TODOS.md. Persistent knowledge store.

AI Assistants
Chief of Staff

Strategic Orchestrator

The primary assistant that transforms Ideas into Projects and Plans, aligns work to Goals, and maintains the system's memory.

  • Full document read/write via Contentbase
  • Transforms Ideas → Projects → Plans
  • Maintains SELF.md, USER.md, TODOS.md
  • Reports progress and system health
  • Routes work to AI coding assistants
Voice Assistant — "Friday"

Voice-First AI

Irish-accented conversational AI for voice-to-voice interaction. Brief responses (1-5 sentences), designed for spoken output.

  • ElevenLabs TTS with expressive tags
  • No markdown, no code — all spoken
  • Command dispatch via voice handlers
  • Wake word: "Hey Friday"
  • Conversational fallback mode
Task Scheduler
Agentic Loop Engine

Polls docs/tasks/ and docs/plays/ every N minutes. The heartbeat of the autonomous system.

Scheduling Logic
  • Filters Plays by schedule (daily, 4pm, hourly)
  • Filters Tasks by: not yet run, not in-progress
  • Max concurrent execution control
  • Lock-based double-execution prevention
Execution
  • Runs via luca prompt agent task-id
  • Updates task meta with lastRanAt
  • Captures output to logs/prompt-outputs/
  • Configurable tick interval (default: 1 min)

Component Details
  • Rustpotter — Wake word detection with trained .rpw models
  • VoiceListener — Audio capture with timeout-based locking
  • Whisper MLX — Local speech-to-text transcription
  • VoiceRouter — Routes to handlers or conversational fallback
  • Voice Handlers — Custom modules matching command utterances
  • VoiceChat — Wraps Assistant + SpeechStreamer for TTS
Presenter
Native macOS App — LucaVoiceLauncher

Window Manager

Swift-based native app that opens and manages browser windows and PTY-backed terminal windows. Communicates via Unix domain socket IPC with NDJSON protocol.

  • HTTP server on port 9210
  • WebSocket on port 9211
  • Window lifecycle notifications
  • Dual role: Dock app + login agent
