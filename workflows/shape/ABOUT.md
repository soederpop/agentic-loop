# Shape

Takes a raw idea (spark or exploring status) and walks the user through a structured interview to develop it. Covers problem definition, scope, dependencies, success criteria, effort, and priority. The outcome is a decision: mark the idea as ready, keep exploring, or park it.

## When to use

Use this when the user wants to develop an idea that's been captured but not yet fully thought through. This is the bridge between "I had an idea" and "this is ready to become a project."

## Trigger signals

- "Let's flesh out this idea"
- "I want to develop [idea name]"
- "What about the [idea name] idea?"
- "Let's shape something"
- "I want to work on an idea"
- "Can we talk through [idea name]?"
- The user references a specific idea by name and wants to think it through
- The user asks what ideas need attention — and there are ideas in spark/exploring status

## When NOT to use

- If the user just wants to jot down a new idea quickly — use **Capture** instead
- If the idea is already at ready status — it may be time to promote it to a project instead
- If the user wants to review all ideas at a glance — use **Status Briefing** instead
