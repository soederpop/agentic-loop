# Capture

A lightweight interface for quickly jotting down ideas as they come. The user picks a goal to align the idea to, adds tags, and writes a short description. Each idea starts as a `spark`.

## When to use

Use this when the user has an idea they want to record but doesn't need to flesh it out right now. Capture is about speed — get it down before it's lost.

## Trigger signals

- "I have an idea"
- "Let me jot this down"
- "I want to capture something"
- "New idea for..."
- The user describes something that sounds like an idea but hasn't been recorded yet
- During a conversation where the user is brainstorming and wants to save one

## When NOT to use

- If the user wants to develop or flesh out an existing idea — use **Shape** instead
- If the user is reviewing what ideas already exist — use **Status Briefing** instead
- If there are no goals yet — use **Blank Slate** first so the user has goals to align ideas to
