# Project Reviewer

An interactive review session for a specific project. Shows the project document and all its plans, then opens a chat with an assistant (chiefOfStaff) who gives a briefing on the project's state, flags concerns, and discusses next steps. This is the "let's sit down and review this project together" workflow.

## When to use

Use this when the user wants a focused, conversational review of a specific project — not just a status glance, but a discussion about what's working, what's stuck, and what should happen next.

## Trigger signals

- "Let's review [project name]"
- "What's the state of [project name]?"
- "I want to go over the [project name] project"
- "Review my project"
- "What should we do next on [project name]?"
- "Are there any issues with [project name]?"
- "Let's talk about [project name]"
- The user has a project with multiple plans and wants an opinionated assessment

## When NOT to use

- If the user wants a quick overview of all projects — use **Status Briefing** instead
- If the user hasn't created any projects yet — there's nothing to review
- If the user wants to capture an idea, not review a project — use **Capture** instead
