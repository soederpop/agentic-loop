# System Setup

A diagnostic dashboard that scans the user's system for required and optional capabilities. Shows what's installed, what's missing, and how to fix each issue. Covers the runtime, API keys, voice pipeline, native app, and the authority process.

## When to use

Use this when the user is setting up for the first time, troubleshooting why something isn't working, or wants to verify their environment is complete. This is the "is everything installed and configured?" workflow.

## Trigger signals

- "Is my system set up correctly?"
- "Why isn't voice working?"
- "What do I need to install?"
- "Check my setup"
- "Something's broken"
- "I just installed [tool], did it work?"
- "What API keys do I need?"
- The user reports an error that could be caused by a missing dependency or key
- A new user is onboarding and needs to configure their environment

## When NOT to use

- If the user wants to set up their vision, goals, and ideas — use **Blank Slate** instead
- If the system is already fully configured and the user has a different problem
