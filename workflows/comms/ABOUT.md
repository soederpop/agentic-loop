# Communications Hub

A unified management interface for all communication channels. Install dependencies, configure per-channel settings, test connectivity, and set up rules for how assistants react to incoming messages.

## When to use

Use this when the user needs to set up, configure, or troubleshoot any communication channel (iMessage, Telegram, Google Workspace), or when they want to configure how assistants respond to incoming messages across channels.

## Trigger signals

- "set up comms"
- "configure telegram"
- "configure imessage"
- "set up google workspace"
- "communications setup"
- "who handles messages"
- "message routing"
- "test imessage"
- "channel status"
- "set up telegram bot"
- "add a trusted sender"
- "reaction rules"

## When NOT to use

- If the user wants to actively chat with an assistant — use the assistant-designer or project-builder workflows
- If the user wants to check general system setup (bun, API keys, voice) — use the setup workflow
- If the user just wants to send a quick message — use the imsg/gws features directly
